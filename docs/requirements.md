# 요구사항 명세서

## 1. 기능 요구사항

### 1.1 거래 기록 관리

#### 기능 개요
- 수입 / 지출 내역 등록 기능 제공
- 거래 유형(수입 / 지출) 선택 가능

#### 입력 항목
- 날짜 (필수)
- 금액 (필수)
- 카테고리 (필수)
- 메모 (선택)
- 거래 유형 (필수)

#### 입력값 검증
- 날짜는 유효한 날짜 형식이어야 한다
- 금액은 숫자이며 0보다 커야 한다
- 카테고리는 비어 있을 수 없다
- 거래 유형은 반드시 선택해야 한다

#### 저장 기능
- 입력값 검증 후 데이터 저장
- 저장 성공 시 목록 및 요약 데이터 갱신
- 저장 실패 시 오류 메시지 표시

---

### 1.2 거래 내역 조회

#### 기능 개요
- 전체 거래 내역 리스트 조회 기능 제공

#### 조회 항목
- 날짜
- 금액
- 카테고리
- 거래 유형
- 메모

#### 필터링 기능
- 기간별 조회 (일/주/월/사용자 지정)
- 카테고리별 조회
- 거래 유형별 조회

#### 정렬 기능
- 날짜 기준 (오름차순 / 내림차순)
- 금액 기준 (오름차순 / 내림차순)

- 필터 조건은 복합 적용 가능해야 한다

---

### 1.3 기간별 재정 요약

#### 기능 개요
- 연 / 월 / 주 기준 재정 요약 정보 제공

#### 제공 정보
- 총 수입
- 총 지출
- 순이익 (총 수입 - 총 지출)

#### 기간 선택 방식
- 연별 조회
- 월별 조회
- 주별 조회

#### 기본 동작
- 기본 조회 기준은 현재 날짜를 포함하는 기간으로 설정한다
    - 연별: 현재 연도
    - 월별: 현재 월
    - 주별: 현재 주

#### 집계 규칙
- 선택한 기간에 포함된 거래 내역만 집계 대상으로 반영해야 한다
- 수입 거래는 총 수입에 합산한다
- 지출 거래는 총 지출에 합산한다
- 순이익은 총 수입 - 총 지출로 계산한다

#### 추가 정보
- 카테고리별 지출 통계 제공

#### 데이터 갱신
- 거래 내역 추가, 수정, 삭제 시 요약 데이터가 즉시 갱신되어야 한다

#### 빈 상태 처리
- 해당 기간에 거래 내역이 없는 경우
    - 총 수입 0
    - 총 지출 0
    - 순이익 0
    - 빈 상태 메시지 표시

### 1.4 시각화 기능 (선택)

#### 기능 개요
- 소비 및 재정 흐름을 시각적으로 확인할 수 있어야 한다

#### 제공 기능

- 기간별 수입 / 지출 추이 시각화 (바 차트)

### 1.5 예산 관리

#### 기능 개요
- 월별 예산 설정 및 수정 기능 제공

#### 제공 기능
- 월별 예산 설정 기능 제공
- 예산 수정 기능 제공

#### 예산 현황 정보
- 현재 지출 금액
- 남은 예산
- 예산 소진율

---

### 1.6 예산 경고 알림

#### 기능 개요
- 예산 사용량에 따른 경고 메시지 제공

#### 경고 기준
- 예산 80% 이상 사용 시: 주의
- 예산 90% 이상 사용 시: 경고
- 예산 초과 시: 초과 알림

#### 동작 조건
- 거래 등록 시 예산 상태 반영 알림 가능

---

### 1.7 데이터 저장

#### 저장 대상
- 거래 데이터 저장 및 유지
- 예산 데이터 저장 및 유지

#### 유지 조건
- 새로고침 이후에도 데이터 유지

## 2. 비기능 요구사항

### 2.1 UI/UX

#### UI 요구사항
- 반응형 웹 지원 (모바일 / 태블릿 / PC)
- 주요 기능은 3클릭 이내 수행 가능해야 한다
- 직관적인 UI 제공
- 데이터가 없는 경우 빈 상태 화면 제공

---

### 2.2 성능

#### 성능 요구사항
- 필터 및 정렬 시 빠른 응답 속도 유지
- 대량 데이터에서도 안정적인 렌더링

---

### 2.3 데이터 관리

#### 데이터 관리 요구사항
- json-server 기반 데이터 저장
- 데이터 영속성 보장

---

### 2.4 오류 처리

#### 오류 처리 요구사항
- 서버 통신 실패 시 오류 메시지 제공
- 잘못된 입력은 저장되지 않도록 처리
- 로딩 상태 표시
